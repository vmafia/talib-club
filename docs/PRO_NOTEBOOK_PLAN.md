# โปรเจกต์: Ultimate Pro Notebook สำหรับ Talib Club 🚀

เอกสารนี้สร้างขึ้นเพื่อให้ AI ในเครื่องอื่นสามารถรับช่วงต่อการทำงานได้ทันที โดยสรุปแผนงานและสิ่งที่ทำไปแล้วไว้ที่นี่

## 🎯 เป้าหมาย (The Goal)
สร้างระบบสมุดโน้ตระดับ Pro (สไตล์ GoodNotes + Miro) ในหน้าห้องอ่านหนังสือ เพื่อให้ผู้ใช้สามารถวาดรูป, ไฮไลต์, แปะรูปภาพ, วาง Post-it และจดโน้ตได้

---

## ✅ สิ่งที่ทำเสร็จแล้ว (Phase 1: Completed)
- ติดตั้ง `tldraw` ลงในโปรเจกต์
- สร้างคอมโพเนนต์ `ProNotebook.jsx`
- ปรับแต่งหน้าจอ `ReadingApp.jsx` ให้เป็น Split Screen (แบ่งซ้าย-ขวา) บนจอคอมพิวเตอร์และแท็บเล็ต
- ซ่อนการใช้งานบนหน้าจอมือถือ (บังคับใช้ฟอร์มจดข้อความแบบเดิม)
- ปรับ Layout ให้สมุดโน้ตกว้างขึ้น (55%) และซ่อน TimerPanel ชั่วคราวเมื่อเปิดสมุดโน้ต เพื่อลดความอึดอัดของหน้าจอ
- ตั้งค่าให้ `tldraw` บันทึกข้อมูลลงในเครื่อง (Local Persistence) แยกตาม ID ผู้ใช้และ ID หนังสือ

---

## 🚧 สิ่งที่ต้องทำต่อไป (Pending Phases)

### Phase 2: The Smart Annotation (เขียนทับหนังสือ + Export PDF)
- **ระบบถามก่อนใช้:** เมื่อเปิดสมุดโน้ต ให้ระบบถามผู้ใช้ว่า "จดสมุดเปล่า" หรือ "เขียนทับหนังสือ"
- **Automated PDF Background:** หากผู้ใช้เลือกเขียนทับหนังสือ ให้ดึงไฟล์ PDF ของหนังสือเล่มปัจจุบันมาแปลงเป็นรูปภาพพื้นหลังให้ `tldraw` โดยอัตโนมัติ (ปัจจุบันใช้ iframe ทำให้วาดทับไม่ได้ ต้องรื้อเป็น Native PDF Reader เช่น `react-pdf` หรือแปลง PDF เป็นรูปลงกระดาน)
- **Export to PDF:** สร้างระบบส่งออกหน้าสมุดโน้ตที่วาดทับหนังสือออกมาเป็นไฟล์ PDF

### Phase 3: The Audio Note (ระบบบันทึกเสียงลงกระดาษ)
- เขียน Custom Shape สำหรับ `tldraw` ให้เป็น "สติกเกอร์อัดเสียง"
- เมื่อกดอัดเสียงและหยุดอัด ให้เซฟไฟล์เสียงลง Firebase Storage และแสดงเป็นไอคอนลำโพงบนหน้ากระดาษ
- สามารถกดฟังเสียงซ้ำจากไอคอนบนกระดาษได้เลย

## 🤖 คำแนะนำสำหรับ AI บนเครื่องใหม่
เมื่อคุณ (AI) ได้อ่านเอกสารนี้แล้ว:
1. โปรเจกต์อยู่ในสถานะ Phase 1 เสร็จสมบูรณ์ โค้ดทั้งหมดรันได้จริง 
2. ให้รอรับคำสั่งจากผู้ใช้ (User) ว่าจะให้เริ่มลงมือทำ **Phase 2 (เขียนทับหนังสือ/แปลง PDF เป็นพื้นหลัง)** เลยหรือไม่
3. คุณสามารถเข้าไปตรวจสอบโค้ดล่าสุดได้ที่ `src/pages/ReadingApp.jsx` และ `src/pages/reading/components/ProNotebook.jsx`

## 📋 แผน Tablet UX — 11 ข้อ (จัดกลุ่ม A–D)
บันทึกจากแชตที่ทำบนอีกเครื่อง (Huawei) เพื่อไม่ให้แผนหายอีก ลำดับทำ: A → B → C → D

### 🟢 กลุ่ม A — ความชัดเจน/flow (ปลอดภัย) — ✅ เสร็จแล้ว (commit 55a849b)
- **ข้อ 3+11:** แจ้งทุกครั้งว่าจะเปิด "หน้าเปล่า" หรือ "อิมพอร์ต PDF" + ป้ายบอกว่า PDF ข้างๆ ขีดเขียนไม่ได้ (ต้องอิมพอร์ต)
- **ข้อ 6:** แทรกอิโมจิตรงจุดที่ซูมอยู่ (กลางจอปัจจุบัน) ไม่ใช่ตำแหน่งตายตัว
- **ข้อ 8:** ปรับเลเยอร์ขึ้น/ลงได้ทุกวัตถุ
- **ข้อ 10:** ปุ่มลบ/ยางลบไม่ให้กดโดนง่าย (eraseObjects ปิดเป็นค่าเริ่มต้น)

### 🟡 กลุ่ม B — Layout แท็บเล็ต (ต้องผู้ใช้ช่วยเช็คบนแท็บเล็ตจริง) — ✅ ลงโค้ดแล้ว รอทดสอบ
- **ข้อ 1+4:** แถบล่างโดนแท็บเบราเซอร์บัง → แก้ด้วย `.reading-room-shell { height: 100svh }` (fallback 100vh) ใน `global.css` + เปลี่ยน `inset:0` เป็น `top/left/right:0` ใน `ReadingApp.jsx` + เพิ่ม hint "ติดตั้งเป็นแอป" (ใช้ `usePWA` hook) โชว์ในห้องอ่านตอนยังไม่เปิดโน้ต

### 🔴 กลุ่ม C — Gesture/ปากกา (ต้องลองบนแท็บเล็ตจริงเป็นรอบๆ) — 🟡 แก้เท่าที่ปลอดภัยแล้ว
- **ข้อ 7:** ✅ แก้แล้วแบบไม่แตะ gesture logic — เพิ่ม hint ตอนเข้าโหมดบ่วงว่า "สองนิ้วเลื่อน/ซูมได้เลย" (ความสามารถนี้มีอยู่แล้ว `handlePinch` ทำ pan+zoom ทุก tool ไม่ผูกกับ tool)
- **ข้อ 5:** ⏳ ต้องเทสบนแท็บเล็ต — `touchAction:'none'` มีบน Konva Stage แล้ว ([ProNotebook.jsx:3551]) การเหมา touch-action ทั้ง container จะพัง touch-scroll ของ toolbar จึงห้ามทำ; การติดตั้งเป็นแอป (standalone จากกลุ่ม B) ช่วยลดการซูมทั้งหน้าเว็บได้ในตัว
- **ข้อ 1:** ⏳ palm-rejection / ปากกาหน่วง = จูนด้วยความรู้สึกล้วน ต้องลองบน Huawei ทีละรอบ ห้ามเดาแก้
- หมายเหตุ: โค้ดมี palm-rejection + pinch-zoom อยู่แล้ว ข้อ 1/5 ต้องมีแท็บเล็ตในมือจึงจะจูนได้ถูก

### 🔵 กลุ่ม D — งานใหญ่ — ⏳ ยังไม่ทำ
- **ข้อ 2:** แคป PDF เลื่อนไม่ได้ + ดึง PDF ใหม่ทุกครั้ง → ต้องแคชไฟล์ + ทำ modal เลื่อนได้ (งานกลาง)
- **ข้อ 9:** จัดรูปแบบเฉพาะบางบรรทัดในกล่องเดียว (bullet แค่ 2 บรรทัด) → ต้องรื้อระบบข้อความเป็น rich-text (งานใหญ่สุด)
